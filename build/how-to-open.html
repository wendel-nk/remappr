“Apple could not verify…” — how to open Remappr

Drag Remappr into Applications first. The first launch is blocked by Gatekeeper because Remappr is an open-source app without an Apple Developer ID, so it is not notarized. The app is safe — you only need to allow it once.

macOS 15 (Sequoia) and later

  1. Double-click the app once. In the “could not verify” dialog choose Done (not “Move to Trash”).
  2. Open System Settings → Privacy & Security, scroll down to the Security section — it says “Remappr was blocked…”. Click Open Anyway.
  3. Confirm with Touch ID or your password. Done — this is only needed once.

macOS 14 (Sonoma) and earlier

Right-click (or Ctrl-click) the app in Applications, choose Open, then Open again in the dialog.

Any version — Terminal

xattr -dr com.apple.quarantine /Applications/Remappr.app

Why does this happen? Removing the warning entirely requires a paid Apple Developer ID and notarization, which this free open-source project does not have. The build is code-signed ad-hoc and published openly at github.com/Wolffyx/remappr.
